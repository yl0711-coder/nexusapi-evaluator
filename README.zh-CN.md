# NexusAPI Evaluator

私有仓库。该项目是内部专有工具，不是开源项目。

NexusAPI Evaluator 是一个本地模型与渠道评测工具原型。

当前版本是本地桌面 MVP：前端使用 Vite，桌面壳使用 Tauri，本地 API 服务暂时使用 Node.js。它的目标是先把配置录入、测试流程、本地日志、报告产出和非技术人员操作流程跑通，再逐步升级为正式桌面工具。

## 当前能力

- 管理评测 API 配置，包括目标 API 和裁判 API。
- 将 API 配置元数据保存在 `NexusAPI数据/配置/profiles.json`。
- 支持 OpenAI-compatible 和 Claude Messages 两类协议的快速连通性测试。
- 支持一键标准评测：快速测试、低轮稳定性测试和少量复杂场景初筛。
- 支持本次测试信息记录，交付模板会自动带上项目、批次、测试人员和测试目的。
- 支持多轮稳定性测试，可配置轮数和轻量并发。
- 支持多个 API 配置的批量稳定性测试，不需要打开多个窗口。
- 支持连通性、速度、结构化输出、代码排错、长上下文、推理决策、商务写作等场景测试。
- 提供操作模板、口语化成本预估、进度展示和长任务取消。
- 提供 API 配置模板、场景包、错误处理建议和标准评测下一步建议，降低非技术人员操作成本。
- 支持导出/导入 API 配置模板，导出内容不包含 API Key，导入后会提示缺 Key 配置并引导补 Key。
- 支持页面内短帮助、空状态引导、演示模式和失败后的具体操作按钮，适合非技术人员按流程使用。
- 将请求指标记录到 `NexusAPI数据/日志/requests.jsonl`。
- 将稳定性测试摘要保存到 `NexusAPI数据/日志/test-runs.jsonl`。
- 在 `NexusAPI数据/报告/` 生成 Markdown 和 HTML 报告。
- 展示请求状态、耗时、token 用量、输出摘要、错误类型、成功率、慢请求参考和推荐结论。
- 在报告中心展示模型/渠道排行榜，按成功率、慢请求参考和场景质量分做快速横向对比。
- 保存 API 配置前会做基础检查，阻止明显错误的 Base URL，并提示协议、超时和输出长度风险。
- 标准评测完成后显示人话结论和下一步按钮，方便非技术人员继续操作。
- 前端只展示非技术用户能理解的错误提示和错误编号，技术细节写入本机 `NexusAPI数据/日志/errors.jsonl`。
- 支持“保存并测试配置”、报告中心“极简结论”和“一键导出问题包”，减少普通操作员的判断成本。
- 日志和报告只保存在本机，API Key 不写入请求日志或报告，常见敏感字段会在日志和报告中脱敏。
- 长时间测试时会裁剪 JSONL 日志、限制上游单次响应大小，并且任务完成后只在内存保留摘要和报告路径。

## 启动本地 Web 页面

```bash
pnpm dev:server
pnpm dev
```

然后打开：

```text
http://127.0.0.1:5179
```

## 启动桌面壳

```bash
pnpm dev:desktop
```

启动普通版或内部风控版：

```bash
pnpm dev:desktop:standard
pnpm dev:desktop:risk
```

普通版会隐藏内容安全合规测试；内部风控版会显示内容安全合规测试。

这个命令会打开桌面窗口。工具需要的本机通信通道会自动处理，普通使用者不需要理解或设置端口。

如果本机通信通道被其他软件占用，工具会自动换一个可用通道，不会关闭任何已有程序，也不会影响 VPN、代理、数据库等本机服务。只有自动处理失败时，才会提示“请重新打开工具，仍失败则把提示发给负责人”。

维护人员如需查看端口细节，可设置 `SHOW_TECHNICAL_PORT_DETAILS=1` 后启动。

## 基础操作流程

1. 使用 `pnpm dev:desktop` 打开应用。
2. 在 `API 配置` 里添加目标 API。
3. 回到 `总览` 填写本次测试信息，后续会自动写入交付模板。
4. 普通操作员优先运行 `标准评测`，一次完成连通、低轮稳定性和少量场景初筛。
5. 在 `报告中心` 查看极简结论、排行榜、测试记录和请求日志。
6. 进入 `测试交付` 复制 Markdown 交付模板，或打开 `NexusAPI数据/报告/` 下保存的报告文件。

高级复测入口：

1. 需要手动排查时，运行一次 `快速测试`，确认 URL、Key、模型名和协议正确。
2. 需要扩大轮数时，运行 `稳定性测试`：
   - `3 轮` 用于冒烟测试。
   - `10 轮` 用于基础稳定性测试。
   - `30 轮` 用于较完整的稳定性检查。
3. 对比多个模型或渠道时，使用 `批量并发测试`：
   - `同时测试 API 数` 控制同时测试多少个配置。
   - `单 API 请求并发` 控制单个配置内部的请求并发。
4. 评估复杂任务能力时，使用 `场景测试`：
   - 可以使用低成本初筛包、编程能力包、长上下文包、基础全量包或候选深度复测包。
   - 初筛建议先用低成本初筛包。
   - 严肃对比候选渠道时再提高重复次数。

## 内置场景类型

- 连通性：确认 API 能正常返回。
- 速度：测试短请求延迟和基本完整性。
- 结构化 JSON：检查模型是否能遵循严格 JSON 输出要求。
- 代码排错：检查代码分析、修复方案和测试思路。
- 长上下文：检查长输入下的摘要和风险提取能力。
- 推理决策：检查多因素业务决策推理能力。
- 商务写作：检查面向非技术用户的表达能力。

## 安全说明

API Key 不再写入 `NexusAPI数据/配置/profiles.json`。

在 macOS 上，优先使用系统 Keychain 保存 Key。其他系统或 Keychain 不可用时，会退回到 `NexusAPI数据/.vault/` 下的本地加密 vault。

报告、请求日志、导出配置和交付模板都不包含 API Key。

## 报告结构

生成的 Markdown / HTML 报告采用双层结构：

- 第一层是“给业务人员看的结论”，只说明是否推荐继续、主要原因和下一步。
- 第二层是“专业分析明细”，保留成功率、慢请求参考、错误分布、单轮明细、token 用量和技术诊断，方便技术人员排查。
