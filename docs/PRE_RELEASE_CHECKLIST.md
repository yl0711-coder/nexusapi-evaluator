# NexusAPI Evaluator 打包前验收清单

本文档用于每次正式打包前检查。普通外包版和内部风控版都要按清单验收。

## 1. 版本模式确认

普通版：

```bash
pnpm dev:desktop:standard
```

- 首页顶部显示“普通版”。
- 场景测试里看不到“内容安全合规包”。
- 标准评测、快速测试、稳定性测试、能力场景测试仍可正常使用。

内部风控版：

```bash
pnpm dev:desktop:risk
```

- 首页顶部显示“内部风控版”。
- 场景测试里可以看到“内容安全合规包”。
- 内容安全测试只允许负责人或受训测试人员使用。

## 2. 自动化检查

每次打包前执行：

```bash
pnpm test
pnpm build
pnpm check
```

全部通过后再进入人工流程。

## 3. 真实流程验收

使用真实 API 配置验收：

1. 新增一个 API 配置。
2. 填写 Base URL、模型名、API Key。
3. 点击“保存并测试配置”。
4. 确认快速测试成功，且日志/报告不出现真实 API Key。
5. 进入“标准评测”，跑 3 轮标准评测。
6. 查看“人话结论卡”和 Markdown 详细结果。
7. 进入“稳定性测试”，跑 3 轮冒烟测试。
8. 进入“报告中心”，确认极简结论、排行榜、最近请求、任务记录正常。
9. 进入“测试交付”，复制交付模板。
10. 导出问题包，确认不包含 API Key。

内部风控版额外验收：

1. 进入“场景测试”。
2. 选择“内容安全合规包”。
3. 只选择 1 个 API，重复 1 次，并发 1。
4. 确认报告能体现内容安全风险，不输出违规细节。

## 4. 数据目录验收

确认项目目录下生成：

```text
NexusAPI数据/
  配置/
  报告/
  日志/
  .vault/
  .runtime/
```

验收规则：

- 报告文件在 `NexusAPI数据/报告/`，用户能直接找到。
- 请求日志、任务日志、错误日志在 `NexusAPI数据/日志/`。
- `NexusAPI数据/.vault/` 不要发送给任何人。
- 旧 `app-data/` 如果存在，启动时会复制迁移到新目录；确认新目录正常后再决定是否清理旧目录。

## 5. 打包产物验收

普通版打包：

```bash
pnpm tauri:build:standard
```

内部风控版打包：

```bash
pnpm tauri:build:risk
```

人工检查：

- 双击应用能打开。
- 不影响本机 VPN 或代理端口。
- 关闭应用后端口释放。
- 重新打开后历史配置和报告仍在。
- 普通版不显示内容安全合规包。
- 内部风控版显示内容安全合规包。

## 6. 不允许交付的情况

出现以下任意情况，不允许打包交付：

- 测试未通过。
- 报告或日志出现真实 API Key。
- 普通版仍显示内容安全合规包。
- 启动程序影响用户本机 VPN、浏览器或其他常用软件。
- 报告无法在 `NexusAPI数据/报告/` 找到。
