第 1 步 · API 配置

配置要评测的渠道

普通用户只填必填项，点「保存并测试」。Key 只存本机，不进报告/导出。

添加 / 编辑渠道

* 为必填

拿到一个 API 地址和 Key 就能测。不懂协议先选「AI 中转站 / OpenAI 兼容」模板。

模板只辅助填写，保存前仍可改。
人能看懂的名字，报告里会显示。
复制后台模型名，不要自己猜。
填到域名/网关：https://api.example.com，不要带 /v1/chat/completions。
只保存在本机，不会出现在报告、日志或导出里。
高级设置，一般不用改（角色 / 协议 / 超时 / 成本与售卖单价 / 备注）
✓ 填写配置后，这里会提示明显风险（如 Base URL 带了业务路径、协议可能选错）。

填写要点

Base URL填到域名或网关：https://api.example.com，别把完整业务路径填进去。
协议多数中转站选 OpenAI Compatible；Claude 原生接口才选 Claude Messages。选错常表现为内容块缺失/空响应。
API Key只保存在本机安全存储，导出配置、报告、日志都不带真实 Key。
可信基线官方或长期稳定渠道可设为「可信基线 API」，后续做横向对比和防换模型用。
