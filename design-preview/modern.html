帮助使用手册

不懂怎么测？这页讲清流程、按钮和常见问题。所有数据只保存在本机。
🚀
第一次用？只走这 5 步

配置 API → 准入评测 → 标准评测 → 看报告 → 复制交付。普通操作员不用碰高级测试。

快速开始

在「API 配置」填入平台给的地址、Key、模型名（不懂协议先选"AI 中转站 / OpenAI 兼容"），点 保存并测试配置。通过后回总览，跟着"下一步"提示走即可。

五步流程说明

  • ① API 配置：录入要测的渠道；Key 只存本机、不进报告/导出。
  • ② 准入评测：配置 / 连通 / 协议 / 身份 四道门槛，过不了就停，省额度。
  • ③ 标准评测：一键跑连通 + 稳定性 + 少量场景，拿分级结论。
  • ④ 报告中心：看结论、横向排行、失败原因和报告路径。
  • ⑤ 测试交付：复制交付模板，把结果、异常、附件一次说清。

结论怎么看

结论分 推荐 / 观察 / 不推荐。成功率带 95% 置信区间，延迟看 P95/P99（不看平均）。身份与计费只给"疑似 / 需上游解释"，不写"确定造假"——这是软件黑盒的边界，不是能力不足。

数据与安全

所有数据只保存在本机 NexusAPI数据。API Key 存系统钥匙串；导出配置、报告、日志都不带真实 Key，敏感字段自动脱敏。

常见问题

  • 空响应 / 内容块缺失：多半协议选错，试试 OpenAI Compatible 或直连同协议。
  • 偶发超时：复杂任务把超时调到 60000–120000 ms。
  • 看不懂结论：先看报告第一段"给业务人员看的结论"，再看明细。
